 Abstract

     This memo provides a major revision to my Internet RFC 1394 by:
     including new information which was not available to me when the
     original document was released; changes to old information due to
     the destruction of the Soviet Union, East Germany and Yugoslavia,
     and the Balkanization of Czechslovakia; corrects minor format
     errors; corrects some misunderstandings which occurred from some
     people's misreading of the original RFC document; and removes some
     omissions which occurred in the original RFC.  As the corrections
     (including a whole new set of information which is added to the
     original document) are substantial, the original RFC 1394 will be
     obsoleted rather than revised when this document is published as
     an RFC.

     This memo lists known Internet domains, telex dial and answerback
     codes, country identifiers, X.400 providers, international dialing
     codes, public data network identification codes, time zone(s)
     where available, and major "public" Internet sites.

     This list is designed to show the corresponding codes for Fax and
     voice messages via telephone, data connections, telex country codes,
     telex answerbacks and Internet domains.  It is an attempt to place
     all of the information relating to computer and/or telephone
     communication anywhere in the world into one list and in that single
     list show all the connections for each country or area, including
     places not yet having Internet Service.

 How to use this document

     This document begins with a list of countries, areas or domains, and
     provides additional information about each one.  Next comes the list
     of countries and areas in order by their two- or three-character
     indentifier code.  Following that is the list of DNIC codes in order
     by code.  Finally, a list of International Telephone area codes, in
     order by code, appears at the end of the document  The information
     is as follows:


 Format of List of Countries and Domains

     Name:The first item appearing is the name of each country or area;
     it is presumed to be a string which may contain a blank or comma,
     and is terminated by the first occurance of two blanks, which are
     not considered part of the name.  If the name plus two spaces would
     cause the area code field (or four dashes) which follows it to end
     past column 29, the name field will be blank on the next line.

     Area Code:     Each name is presumed to be followed by the
                    international telephone area code for that area, if
                    available.  If the telephone area code is not
                    available, four dashes appear.  The area code begins
                    at any point after the two blanks following the name,
                    and is right justified so as to end in column 29.  If
                    the "Name" string would not allow the telephone
                    number (or four dashes) to end in column 29, then the
                    telephone area code appears starting in the
                    appropriate column on the next line, and the area
                    name will be blank.  If there is more than one area
                    code for an area, a comma follows in column 30, and
                    the information starts over again on the next line in
                    the appropriate column.

     Telex Code:    The international telex code for that country or
                    area, if available.  It starts in Column 34.  (All
                    entries except US have 3 digits or 3 dashes; the US
                    entry consists of a blank and its two digit number.)
                    If an area has more than one telex number code, it
                    will terminate with a comma in column 37, and the
                    next entry will begin on another line in column 34.

     Telex 
     Answerback:    A one, two, or three letter code indicating a
                    particular area.  Note that the geographic answerback
                    on a telex is not always reliable since some places
                    do not give a standard geographical answerback and
                    some duplicate answerbacks are used by multiple
                    countries.  The code starts in column 38, and if more

                    than one answerback exists for an area, it is
                    followed by a comma, and the next entry appears in
                    column 38 on the next line.

     F.69 Telex 
     Code:          The F.69 telex code used for areas, where available. 
                    This set of codes is included because some telex
                    systems use it in place of the above international
                    telex codes, or have F.69 as an option. It is a 2- or
                    3-digit number left justified to start in column 42. 
                    If an area has more than one F.69 code, the F.69 code
                    is followed by a comma.  If all the F.69 entries for
                    an area will not fit on a line, they are continued
                    starting in column 42 on the next line.

     Time:          This is a special 1-Character field which indicates
                    the basic time zone for this region of the world as
                    defined generally in RFC 842.  It appears in column
                    46 UNLESS there is no space between itself and the
                    F.69 telex code, in which case it appears on the next
                    line.   It is a single character.  Where the time
                    value is unknown or if it is undetermined,  a -
                    appears. A Cross-Reference table of the single code
                    appears at the end of the document.

     Domain Code:   For countries and areas, this is the two-letter code
                    assigned to it by the UN ISO 3166 agency, and always
                    starts in column 48.  For Internet, a two-letter code
                    is the Base Domain that is used by that country or
                    geographic area for geographically assigned Internet
                    hosts.  If that country has no Internet hosts (or
                    none that are geographical based), this is the code
                    that {would} be used.  A three-letter code is used
                    exclusively by internet to indicate a "non-
                    geographical" domain.  If an area has more than one,
                    it is followed by a comma.  If a domain name is
                    longer than 3 characters, it is the domain name of a
                    gateway or Internet service provider. 

     ISO 3166 
     Number:        This is similar to the domain code, but is a
                    numerical identifier for countries, always appears in
                    column 52; a country or area can have only one.  The
                    ISO 3166 fields are the ONLY ones that are blank if
                    missing.

     ISO 3166 
     Code:          This is similar to the domain code, but is a 3-letter
                    code assigned to the country or area, always appears
                    in column 56; a country or area can have only one.

                    The ISO 3166 fields are the ONLY ones that are blank
                    if missing.

     DNIC:          "Data Network Identification Code".  This is a code
                    used to connect to various datagram, packet or value-
                    added networks.  It starts at the first non-blank at
                    or beyond column 60.  The code is 4 digits for a
                    network, and is akin to the area code for telephones
                    and telex country code.  DNIC Codes which are longer
                    than 4 digits are the exact network address of a
                    specific computer, which is equivalent to a specific
                    telephone number.  Where an area or name has more
                    than one, the DNIC code is followed by a comma.

     All entries are column alligned.  Where an entry ends with a comma,
     it means that additional entries appear either immediately after the
     comma or at the usual starting column for that entry on the
     following line; if a comma is followed by a space, then the next
     entry appears on the following line for that entry.  Four dashes
     appear where no telephone code is known for an area; three dashes
     appear for all other missing or unknown information except ISO 3166
     (3 Digit and 3 Letter Country) codes; where there is no code for an
     area or it is unknown, the ISO 3166 field is blank.  When a field
     has run out of data for an entry before another field for that same
     entry is finished, the first entry is repeated for all subsequent
     lines.

     A line is continued from a previous line if the name field is blank;
     any field which does not have data appearing to the left of it which
     is identical to the entry it continues, may also be blank.


     Format of International Area Code List

     Country Code:  The country code indicates the 1 to 3 digit
                    identifier which states approximately where a country
                    is.  If a country is listed with more than 3 digits,
                    it means that it is either sharing an international
                    code number with one or more countries; this is
                    usually done for political or technical reasons. 
                    (San Marino, Vatican City and Dominican Republic are
                    examples of countries that share codes with other
                    countries; political subdivision examples are Moscow,
                    Russia; Paris, France; and Wisconsin, USA).   Note
                    that the areas of the world are not exact; for
                    example, Greenland has an "Africa" code of 299
                    because all of the Europe codes were in use.  Note
                    that some of the countries in 1809 may be split off
                    into new international codes if they become too large
                    for the area.

     Country:       The country or area that is using the particular code
                    is assigned to.  Note that some codes are "non
                    geographic" and may be assigned as needed; the
                    Inmarsat codes are used for planes in the air and
                    ships at sea as well as international cellular
                    telephones; the code 1800 is used for calls from
                    within International Zone 1 at no cost to the caller;
                    some of these 1800 numbers terminate in points
                    outside of Zone 1 just as some countries have
                    internal numbers that are no cost to a caller in that
                    country and terminate outside of that country.

 Content, Corrections and Connectability

     The information in this document was obtained from several sources
     in multiple countries; while reasonable care was used in preparing
     this document, its accuracy cannot be guaranteed; in some cases the
     codes may not be correct.  We welcome corrections as well as
     additional countries/zones/areas/major mailing systems/related
     information not listed.  Please feel free to widely distribute this
     list.

     Comments, suggestions and corrections are welcomed; much of the new
     information in this document came from E-Mail from readers of the
     prior document.  Corrections may be mailed, E-mailed or telexed to
     the address at the end of this document.

     Note that whether you can reach a particular country depends on
     whether there is a connection to that place and whether connections
     are permitted from your place.  The listing of a country in this
     document does not guarantee there is any connection on the internet
     to it at all, or if it has one, it may be nothing more than a dial-
     up UUCP or other connection for mail and/or news only.  Note that a
     listing in this document does not mean that you can reach the
     country at all; a country can lose or discontinue its internet
     connections or be cut off.

     The inability to contact an area is often due to political reasons. 
     For example, until recently US callers could not make direct-dial
     telephone calls to Cuba or Vietnam (this should change by the time
     this document is released), and most Arab countries cannot dial
     Israel (the recent change of administration in the Gaza Strip may
     change this).  Other countries have other restrictions, may not have
     any service at all, or may have been disconnected in order to comply
     with sanctions imposed by International Organizations.  In some
     cases and depending on what has happened in the world, telephone or
     telex service may be available while Internet connections are not,
     or vice versa, or none of these may be available.

     No standing as to the validity of a country's name or existence is
     made by this document; this is simply an attempt to catalog all
     known related information.  Information is solicited for extra, old
     or alternate names for any existing country, or any missing details.

     Some entries may or may not be reachable via their ISO 3166 country
     code as a domain on the Internet, as they were part of some other
     country at one time or another and the new code may not yet be
     completely installed.  Also, the United Kingdom has two Internet
     domain names and Russia continues to use its old Soviet Union domain
     name for the time being.

 Notes:

     BITNET

          Within and when attempting to reach the BITNET community, names
          of the form "System.BITNET" are used. (Within the BITNET domain
          the ".BITNET" extension may be omitted in many cases on
          transfers from one bitnet site to another; for example, the
          Internet Fully Qualified Domain Name "PUCC.PRINCETON.EDU" is
          also the internal BITNET domain "PUCC.BITNET" but is also known
          to BITNET clients as the internal domain "PUCC").  These are
          not registered names in the Domain Naming System (DNS).  To use
          BITNET names in the Internet, one must use the "per cent sign
          hack" to route the mail through an Internet/BITNET gateway. 
          There are aproximately five primary and eight secondary Bitnet
          to Internet gateways around the world, the two largest being
          PUCC.PRINCETON.EDU and CUNYVM.CUNY.EDU.  (Many Bitnet mailing
          lists I get on my account on MCI Mail that come from a Bitnet
          exploder do so through Princeton.)

          The following Bitnet gateway sites are in the United States: 
               CUNYVM.CUNY.EDU, PUCC.PRINCETON.EDU, PSUVM.PSU.EDU,
               CMSA.BERKELEY.EDU, UGA.UGA.EDU, VM.USC.EDU

          The following Bitnet gateway sites are in Europe:
               SEARN.SUNET.SE, AEARN.EDUZ.UNIVIE.AC.AT, FRMOP11.CNUSC.FR,
               IB.RL.AC.UK, VM.GMD.DE, NIC.SURFNET.NL

          The following Bitnet gateway site is in the Middle East:
               TAU.AC.IL

          If someone needed to send a message to Jack Ryan, a user with
          an E-Mail address of JRYAN@CIA.BITNET they might use one of the
          following:

               JRYAN%CIA.BITNET@PUCC.PRINCETON.EDU
               JRYAN%CIA.BITNET@CUNYVM.CUNY.EDU
               JRYAN%CIA.BITNET@SUNET.SE

          (To the best of my knowledge, this site, user and address are
          fictional.  Or at least, I hope they are.)


     UUCP

          Within the UUCP community, names of the form "host.UUCP" are
          sometimes used.  As with BITNET, these also are not registered
          names in the DNS.  To use UUCP names in the Internet, one must
          again use the "per cent sign hack" to route the mail through an
          Internet/UUCP gateway.  You will need to check with your

          correspondent to see how to route to them.  Many of them go
          through UUNET or the University of California at Berkely, and
          some go through other sites.  If a named John Clark has an
          address of JCLARK@CIA.UUCP, you might use something like:

               JCLARK%CIA.UUCP@UUNET.UU.NET
               JCLARK%CIA.UUCP@UCBVAX.BERKELEY.EDU

     ARPA

          For historical reasons, a section of the DNS namespace uses the
          ".ARPA" domain to register the addresses of hosts and networks
          to enable reverse lookups.  All .ARPA sites have "regular" 
          Internet DNS addresses in the usual country domains or in the
          non-geographic  .MIL, .NET, .GOV, .EDU, .COM or .INT domains.


     Special Cases:

     -    Czechslovakia is no longer in existence.  Pieces of the old .CS
          domain (Czech Republic .CZ and Slovakia .SI, among other former
          regions) have new international identification codes; check
          with your correspondent to see what is to be used.

     -    The Soviet Union is no longer in existence, and the pieces of
          it have new international identification codes.  Russia has
          been assigned the ISO 3166 code "RU" but sites there continue
          to use the Internet domain ".SU" so the SU code appears in the
          list, until such time as the country changes over to the
          replacement code.  The following countries used to be part of
          the Soviet Union; some sites in these countries may still use
          the .SU domain until and if they establish geographic Internet
          Service under other domains:

               Armenia, Azerbaijan, Belarus, Estonia, Georgia,
               Kazakhstan, Kirghizistan, Latvia, Lithuania, Moldova,
               Russia, Uzbekistan, Tadjikistan, Turkmenistan, Ukraine

     -    The United Kingdom is assigned the ISO 3166 code of GB, while
          on the Internet, it has the assignment of both .GB and .UK. 
          The .GB code is slowly being brought into service with the
          eventual intent of discontinuing the .UK code.

     -    The "Neutral Zone" (portion of Iraq under UN control) was
          eliminated last year.

     -    Certain Countries have "questioned" telephone dialing codes:

          -    One document states that San Marino is 39 541 and will
               eventually move to 295; a telephone company book has it as

     Note that in the list below, where the column "Internet Domain"
     appears, for a country this refers to the two-letter ISO 3166 code,
     which is the same as the Internet Domain.  Note that if the area in
     question does not have Internet Service, this is the code that
     {would be} assigned if the area ever does.  Three-letter codes
     indicate non-geographical Internet Domains.  Other than for
     historical uses, four-letter Internet Domains have been
     discontinued.

     Subdomains will be listed for states or provinces where a country
     assigns them by geographic area. An entry of a private subdomain
     indicates a "large" internet connected service.  "Large" being any
     service essentially accepting anyone with a means to transmit and/or
     receive data and a minor (or no) fee and having more than 3,000
     persons accessible under that service's Internet subdomain name.  I
     also will list, when I know about them, any large public gateways. 
     (A "large public gateway" is any site that accepts mail from any
     Internet address for delivery to another network or another point,
     including gateways that do mail to news transfers for many groups,
     carries over 1,000 newsgroups, and/or handle more than 50,000
     messages a day.) Information on new large sites and new large
     gateways is solicited from readers of this document for inclusion in
     subsequent revisions.

     To distinguish between the two, a political subdivision will be
     indicated by having the name consist of the 3 letter ISO 3166 code,
     followed by a space, a dash, a space and the name of the
     subdivision.  Subdomains which are network gateways or connection
     services will have the word "clients" in the name.

                     Time Zone Cross Reference Table

 The time zones used in the TIME column are based on the 1-letter
 military code for time zones which was identified in Internet RFC 841,
 with additions to cover zones which add fractions of an hour to their
 times, and to cover areas which use more than one zone. Each number is
 the difference between that letter and UTC.  Where the time zone adds
 1/2 hour to a base letter, that base letter appears in lower case.

                              Letter Codes

          Y +12              R +5                D -4  
          x +11 1/2          q +4 1/2            E -5   
          X +11              Q +4                F -6   
          W +10              p +3 1/2            G -7   
          v +9 1/2           P +3                H -8   
          V +9               O +2                K -9   
          U +8               N +1                L -10   
          T +7               Z +0                M -11   
          s +6 1/2           A -1                N -12   
          S +6               B -2                   
          r +5 1/2           C -3                    

 For example, New York is Zone E; London is Zone Z; Paris is Zone N; the
 center part of Australia is Zone v.

                             Symbolic Codes

 - - Time zone is unknown or cannot be determined
 * - Area has multiple time zones

 ! - Atlantic Ocean; Zones F thru N
 @ - Pacific Ocean; Zones E thru V
 # - Indian Ocean; Zones O thru T

 0 - Australia; Zones U, v, and W
 1 - Indonesia; Zones T thru V
 2 - Canada, All; Zones I thru D
 3 - United States, All; Zone K, Zones H thru E
 4 - Mexico; Zones H thru E
 5 - United States - Idaho and Oregon; Zones H and G
 6 - Canada - Manitoba and Saskachewan; 
     United States - Kansas, Nebraska, North Dakota, South Dakota, Texas;
     Zones G and F
 7 - Canada - Ontario, Yukon; 
     United States - Indiana, Kentucky, Tennessee;
     Zones F and E

                       X.400 Reachability Cautions

  Note the following cautions with respect to X.400 services:

     Message reachability from or to the Internet from any specific X.400
     gateway:

     -    may be nonexistent;
     -    may be asymetric (meaning that they can send mail to the       
          Internet but cannot receive mail from it, or that you can send
          mail to them but they can't reply) or may provide a different,
          "unreachable" substitution address for incoming Internet
          messages as opposed to their ordinary E-Mail address.
     -    may require the use of a gateway (an intermediate X.400
          provider) or a "bang path" listing the routing needed to send
          the message, and require the Internet address to be specified  
          backwards,  e.g. "ATTMAIL!internet!TDR.COM!PAUL", or use the   
          "percent sign hack" e.g. "internet%paul@tdr.com"
     -    may require that you preregister each Internet address that
          will send messages to a recipient with the recipient's X.400
          provider before sending messages, and then require that the
          recipient use an internal mailbox for replies in place of the  
          Internet address.
     -    may charge the recipient for messages received, as well as for 
          messages which are sent, and may impose surcharges for messages 
          going to or coming from the Internet.
     -    may require vendor manual handling on messages to or from the
          Internet.
     -    may require that messages from mailing lists be specially      
          formatted in order to reduce the number of addressees listed in
          the header of a message, or the size of the address list on a
          message be less than 2K, and may bounce as nondeliverable
          messages with address lists longer than 2K even if every
          address on that list is valid.
     -    may require messages be split into small pieces, as small as
          10K or less in some cases, or may have no problem handling
          messages of one or two megabytes.
     -    may require that one of you send a message to the other before 
          the second party can respond.
     -    may be multiples of any of the preceding.

     Don't think these providers are necessarily "picking on" the
     Internet; the same conditions often apply when sending messages from
     one X.400 provider to another.  Some charge the same rate for
     messages to and from other systems as they do for local messages. 
     Some charge surcharges for messages sent to other X.400 systems in

     addition to their own charges.  Some systems will act as if the
     Internet is just another X.400 system connected to them.

     specified depends on the service.  Some provide a specific mailbox
